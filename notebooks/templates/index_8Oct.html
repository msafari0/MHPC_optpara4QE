    ❓ Help & Instructions

    What does this predictor do?

    This tool uses a trained Neural Network model to estimate the time per SCF call for your Quantum ESPRESSO calculations. The prediction is based on both your QE input file and system parameters such as number of electrons, atoms, and CPU resources.

    What inputs do I need to provide?
    • QE input file: A valid Quantum ESPRESSO input file (e.g., scf.in).
    • Pseudopotentials (UPF files): All UPF files required by the input file must be uploaded.
    • System parameters:
      • n_el: Number of electrons in the system
      • n_k: Number of k-points
      • n_cores: Number of CPU cores used

    ⚠️ Make sure the input file and UPFs are consistent (all pseudopotentials referenced in the input must be uploaded).

    Troubleshooting

    If your prediction fails with "An error occurred during QE input + pseudos prediction":

    • Check that all required pseudopotentials were uploaded.
    • Ensure file names match exactly what is referenced in the QE input file.
    • If running on the cloud, the issue may be related to file upload limits or missing files in /tmp.
    • Admins can check logs on the server: 
      tail -f /var/log/nginx/error.log
journalctl -u myflask.service -f
    Tips
    • Always verify your QE input locally before uploading.
    • Use descriptive names for input/UPF files to avoid overwriting.
    • For large test cases, check system memory and disk space on the server.
